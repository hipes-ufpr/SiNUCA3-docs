Detailed Description

Class implementation of the static trace reader.

The x86 based static trace is a binary file having the number of instructions and the instructions themselves of each basic block.

"In compiler construction, a basic block is a straight-line code sequence with no branches in except to the entry and no branches out except at the exit" - Wikipedia "Basic block".

The definition above guarantees that when entering a basic block, the program should execute all of its instructions, except in case of interruption. Therefore, the flow of execution can be recorded only by taking note of the indices of the basis blocks executed and storing each basic block once in the static trace. This approach allows for a smaller dynamic trace.

The StaticTraceFile does not use a buffer, unlike the classes that manipulate the Dynamic and Memory files, and uses mmap for file manipulation instead. This is possible because the static trace tends to be small and mmap may be slightly faster for the matter (it's just the dev trying to justify why he did it this way).
