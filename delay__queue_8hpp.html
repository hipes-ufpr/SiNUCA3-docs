Detailed Description

Queue designed to simulate delay.

The delay queue component is a queue designed to simulate delay existing in real life components. Therefore, it has a delay parameter that corresponds to the waiting time in the queue. Beware that any component has a natural delay of two cycles:

clock #1: component 1 sends message to component 2 clock #2: component 2 receives message and forwards it to component 3 clock #3: component 3 receives message

The delay queue allows messages from multiple components to be received, but any request that exceeds the total capacity of the queue will be discarded, so the throughput parameter must be set carefully.

This component uses a cycles counter to set and track the age of the elements enqueued. If the trace being simulated runs for more cycles than 2^64 - 1 - delay, the queue may not work properly.
